# 4号 → 5号 接口文档

> 本文档说明 API 接口、四个模块的数据格式、前端可以绘制什么图。
> 
> **启动后端**：`cd 4_time && uvicorn server:app --port 8000`
> **Swagger**：`http://localhost:8000/docs`
> **Vue baseURL**：`http://localhost:8000/api`

---

## 一、总体概览

4号实现了 **4 个算法模块 + FastAPI 后端**：

| 模块 | 一句话描述 | 当前数据 |
|------|-----------|:--:|
| 舆情生命周期预测 | 判断事件处于哪个阶段，预测未来走势，提前预警 | 3号 114个事件，63个有完整预测 |
| 传播溯源 | 画出传播链路图，找出谁在推动传播 | 1号 B站数据，2个事件有传播分析 |
| 虚假文本检测 | 逐条文章判真假，双模型路由 | 3号 1,228篇文章，模型85.6% |
| 跨事件因果分析 | 发现事件之间的因果链 | 3号 69个事件，173对显著因果 |

---

## 二、API 接口（可直接调用）

### 2.1 接口清单

| 方法 | 路径 | 前端页面 | 说明 |
|:--|:--|:--|:--|
| GET | `/api/events` | 首页热点看板 | 114个事件摘要 + 跨事件因果 |
| GET | `/api/event/{event_id}` | 事件详情页 | 阶段+预测+传播图+文章判假 |
| POST | `/api/articles/check` | 文章列表/详情 | 批量文章逐条判假 |
| GET | `/api/cross-event` | 因果分析页 | 传递熵+格兰杰因果图 |
| GET | `/api/health` | 调试 | 系统状态 |

### 2.2 启动方式

```bash
cd 4_time
pip install -r requirements.txt
uvicorn server:app --host 0.0.0.0 --port 8000 --reload
```

首次启动自动加载数据并初始化模型（约30秒），后续秒级启动。访问 `http://localhost:8000/docs` 可交互测试所有接口。

## 三、接口返回数据格式

### 3.1 `GET /api/events` — 首页看板

```json
{
  "events": [
    {
      "event_id": "EVT_000058",
      "event_title": "中联油致癌物事件延烧",
      "category": "社会民生",
      "current_stage": "高潮期",
      "current_heat_index": 50,           // 0~100
      "trend_direction": "平稳",          // 上升/平稳/下降
      "warning_level": "yellow",          // none/yellow/orange/red
      "sentiment_distribution": {"positive": 0.1, "negative": 0.7, "neutral": 0.2},
      "article_stats": {                  // 仅当事件有文章时存在
        "total": 96, "fake": 39, "uncertain": 24, "real": 33, "fake_ratio": 0.406
      },
      "propagation_summary": "63个节点（23个信息源，40条扩散评论）",  // 仅2个事件有
      "is_resurgence": false
    }
  ],
  "cross_event": {
    "transfer_entropy_pairs": [           // ★ 主输出：符号化传递熵
      {
        "from_event": "EVT_000058", "from_title": "中联油致癌物...",
        "to_event": "EVT_000015", "to_title": "国防部通报...",
        "te_effective": 0.2874, "p_value": 0.0, "is_significant": true
      }
    ],
    "granger_pairs": [...],               // 辅助：格兰杰因果
    "summary": "在4692对事件中发现了2510对显著因果关系..."
  },
  "global_stats": {
    "total_events": 114,
    "alerts": {"red": 0, "orange": 1, "yellow": 41}
  }
}
```

**前端绘制**：
- 事件卡片列表：按 `current_heat_index` 排序，每个卡片标 `current_stage` + `trend_direction` 箭头
- 预警灯：`warning_level` → none绿 / yellow黄 / orange橙 / red红
- 事件关系图：`cross_event` 的 `transfer_entropy_pairs` → 有向图（节点=事件，边=因果）

```json
  "event_id": "EVT_000058",

    "trend_slope": -0.07,

      "grade": "full",                    // full/basic/minimal/sparse
      "effective_nonzero_points": 7,
      "note": "全功能可用（含预测+预警）"
    }
  },

  "propagation": {                        // ★ 仅2个事件（EVT_000058, EVT_000088）有此字段
    "graph": {                            // → 前端力导向图
      "nodes": [{"id":"N001","name":"食品科学阿祖","symbolSize":15,"category":3,"follower_count":0,"source":"B站","pagerank":0.04}],
      "links": [{"source":"N001","target":"N002"}],
      "categories": [{"name":"普通用户"},{"name":"大V/意见领袖"},{"name":"官方媒体"},{"name":"初始信息源"},{"name":"桥接节点"}]
    },
    "key_nodes": [{                       // → 关键节点卡片
      "account_name":"食品科学阿祖","role":"初始信息源",
      "graph_scores":{"pagerank":0.04,"betweenness":0.05,"out_degree_score":0.06,"composite":0.443}
    }],
    "propagation_depth": 2,
    "counterfactual": {"results": [{"account_name":"ULSOAP","causal_impact":{"nodes_lost_if_removed":4}}]},

  "articles": [{                          // → 文章列表+逐条判假
    "article_id":"NEWS_000003","title":"毒油引发食安危机...",
    "verdict":"疑似虚假","confidence_score":9.0,"fake_probability":91.0,
    "risk_factors":["文本模式'苯骈芘'与虚假信息强相关"],
    "shap_explanation": {"summary":"...","top_words":[...]},  // 仅联合模型时有
    "information_sufficiency":"元数据不足","downgrade_reason":"..."
  }],

  "article_stats": {                      // → 统计面板
    "total":96,"fake":39,"uncertain":24,"real":33,"errors":0,"fake_ratio":0.406
  }

```

**前端绘制**：
- 阶段进度条：`stage_probabilities` 四段横向排列
- 热度大数字 + ↑→↓ 箭头：`current_heat_index` + `trend_direction`
- 趋势预测折线图：`predicted_next_24h` — 实线=predicted_count，浅色带=lower_bound~upper_bound
- 拐点标注：`turning_points` 在折线图上打圆点
- 预警灯：`critical_early_warning.warning_level`
- 四维雷达图：`composite_index_breakdown`
- **有 `propagation` 时**：渲染力导向图（`graph`）+ 关键节点卡片 + 反事实推演表
- **无 `propagation` 时**：传播模块可隐藏或显示"暂无传播数据"

### 3.3 `POST /api/articles/check` — 文章判假

**请求体**：
```json
{
  "articles": [
    {
      "id": "1",
      "title": "官方通报",
      "cleaned_text": "据市卫健委通报，今日新增确诊病例3例...",
      "source": "央视新闻",
      "sentiment_intensity": 0.2,
      "is_verified": true,
      "follower_count": 2000000,
      "forward_count": 25,
      "hours_since_event": 8.0
    }
  ]
}
```

**返回**：
```json
[{
  "article_id":"1","title":"官方通报",
  "verdict":"可信","confidence_score":73.6,"fake_probability":26.4,
  "risk_factors":[],"shap_explanation":null
}]
```

## 四、前端可绘制的图汇总

| 页面 | 数据字段 | 图表类型 |
|:--|:--|:--|
| 首页看板 | `events[].current_heat_index` + `current_stage` + `trend_direction` | 事件卡片列表 + 热度排行榜 |
| 首页看板 | `global_stats.alerts` | 预警分布概览 |
| 首页看板 | `cross_event.transfer_entropy_pairs` | 事件因果有向图 |
| 事件详情 | `lifecycle.predicted_next_24h` | 趋势折线图 + 置信区间浅色带 |
| 事件详情 | `lifecycle.turning_points` | 折线图上的拐点标记 |
| 事件详情 | `lifecycle.stage_probabilities` | 四阶段进度条 |
| 事件详情 | `lifecycle.current_heat_index` + `trend_direction` | 大号数字 + ↑→↓ 箭头 |
| 事件详情 | `lifecycle.composite_index_breakdown` | 四维雷达图 |
| 事件详情 | `lifecycle.critical_early_warning` | 信号灯 (绿黄橙红) |
| 事件详情 | `propagation.graph` | ECharts/D3 力导向图 |
| 事件详情 | `propagation.key_nodes` | 关键节点卡片 |
| 事件详情 | `propagation.counterfactual.results` | 因果影响力排行表 |
| 事件详情 | `articles[].verdict` | 三色标签（绿/黄/红） |
| 事件详情 | `articles[].shap_explanation.top_words` | 原文词级高亮 / 词云 |
| 事件详情 | `article_stats` | 判假统计面板 |

## 五、当前数据覆盖情况

| 数据 | 覆盖 | 说明 |
|:--|:--:|:--|
| 事件 | 114个 | 3号 event_timeseries_result.json |
| 文章 | 1,228篇 | 3号 news_output.json，去噪后 |
| 传播分析 | 2个事件 | 1号 B站数据，EVT_000058（食品安全）、EVT_000088（洪涝灾害） |
| 跨事件因果 | 69个事件 | 时间跨度≥3天的事件参与因果分析 |

**其他112个事件无传播数据，事件详情页 `propagation` 字段为 `null`。前端可据此隐藏传播模块。**

## 六、模型信息

| 模型 | 类型 | 准确率 | 说明 |
|:--|:--|:--:|:--|
| 联合模型 | TF-IDF + 元数据 | 85.6% | 元数据充足时启用 |
| 纯文本模型 | TF-IDF only | 84.6% | 元数据不足时自动切换 |
| 数据集 | CED 清华大学 | 3,381条 | 微博谣言数据集 |
