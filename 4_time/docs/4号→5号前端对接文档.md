# 三、模块 1：舆情生命周期预测

## 3\.1 模块功能

输入事件时序热度数据，输出 4 类核心分析结论：

1. 事件当前传播阶段：潜伏期 / 成长期 / 高潮期 / 衰退期 / 二次爆发

2. 未来 24 小时热度预测曲线，附带 95% 置信区间

3. 提前 4\~6 小时热度爆发预警，规避昼夜低谷误判热度下跌

4. 自动剥离昼夜流量周期干扰，区分真实热度下跌与凌晨自然低谷

## 3\.2 输出结果明细

### 1）阶段标签 \+ 概率分布

示例输出：
当前阶段：高潮期
各阶段概率：潜伏期 2%、成长期 10%、高潮期 82%、衰退期 6%
前端适配：横向进度条，当前阶段高亮展示

### 2）综合热度指数 \+ 趋势方向

示例输出：
综合热度：78（取值区间 0\~100）；趋势：平稳（斜率 \- 0\.02）
综合热度四维加权计算规则：

- 报道量占比 40%

- 情感分歧度占比 25%

- 平台扩散度占比 20%

- 内容互动热度占比 15%
前端适配：大号数字展示热度值，↑/→/↓标识趋势；四维雷达图展示四项权重拆解数据

### 3）未来 24 小时热度预测曲线

示例数据：
\+1h：38\.5 篇，置信区间 \[25\.0 \~ 52\.0\]
\+2h：36\.2 篇，置信区间 \[22\.0 \~ 50\.4\]
……
\+24h：12\.1 篇，置信区间 \[0\.0 \~ 28\.3\]
前端适配：折线图，实线为预测值，浅灰色虚线带为置信上下边界；时间越远置信带宽越宽，体现预测不确定性

### 4）阶段切换变点检测

示例输出：
07\-04 09:00 潜伏期转入成长期（热度谷底）
07\-04 14:00 热度加速冲高至峰值区间
07\-05 20:00 高潮期转入衰退期（热度峰值）
前端适配：趋势图标记圆点拐点，鼠标悬浮展示时间与阶段说明

### 5）临界爆发预警

预警分级：绿色（正常）、黄色（关注）、橙色（预警）、红色（高风险）
示例：
预警等级：橙色
触发原因：方差飙升 2\.8 倍、自相关系数 0\.62，传播系统极不稳定
业务价值：热度尚未明显上涨时提前 4\~6 小时预警爆发风险
前端适配：趋势图增加预警标注线，搭配信号灯标签展示等级

### 6）二次爆发检测

两种输出形式：无二次爆发 / 检测到二次爆发
示例：
检测到二次爆发（置信度 84%）
第一轮峰值：07\-05 02:00（55 篇）
热度谷底：07\-05 18:00（18 篇）
第二轮峰值：07\-06 09:00（48 篇）
反弹倍率：2\.6 倍
诱因：官方回应、当事人发声、新增证据引发二次传播
前端适配：趋势图标注两轮峰谷，增加 “二次爆发” 标签高亮

### 7）昼夜周期校正信息

示例：
已自动剥离日周期波动
流量峰值时段：16:00（下午）
流量低谷时段：03:00（凌晨）
前端适配：小型标签提示 “已校正昼夜流量干扰”

# 四、模块 2：传播溯源分析

## 4\.1 模块功能

输入全量转发节点数据，完成 4 类分析：

1. 定位事件原始发布源头节点

2. 构建完整多层级传播链路拓扑图

3. 基于图算法筛选核心关键节点（不单纯依靠粉丝量）

4. 反事实推演：移除单个节点后整体传播规模损失比例，量化因果影响力

## 4\.2 输出结果明细

### 1）传播链路可视化力导向图

节点规则：节点大小 = PageRank 影响力得分；节点颜色区分 5 类用户角色（普通网友、搬运扩散者、大 V、官媒、源头节点）
连线规则：线条代表转发 / 引用关系
示例拓扑逻辑：
源头普通网友→搬运账号、观察员→百万粉大 V（跨圈层桥接核心）→官媒、地方账号→资讯频道
前端适配：ECharts/D3 力导向图，绑定 nodes、links、categories 数组，symbolSize 控制节点尺寸

### 2）核心关键节点评分卡片

综合评分由 PageRank、介数中心性、出度分加权计算，介数中心性越高代表信息流必经节点
示例：
\#1 大 V 李教授
角色：跨圈层桥接节点
粉丝：50 万
PageRank：0\.83 ｜介数中心性：1\.00 ｜出度分：1\.00

\#2 新华网官方账号

综合得分：0\.63
业务说明：粉丝量更高的官媒排名靠后，因为全部传播链路必须经过该大 V，结构影响力更强
前端适配：侧边列表卡片，展示四维评分指标

### 3）反事实因果影响力排行榜

模拟移除节点后传播流失规模，直观对比粉丝量与实际传播影响力
示例：

传播损失：575\.5 万人次，损失比例 99\.9%
原因：所有传播链路必经该账号，无该节点大 V、官媒无法获取信息

\#2 大 V 李教授（50 万粉丝）
传播损失：575 万人次，损失比例 99\.9%

\#3 新华网（500 万粉丝）
传播损失：520 万人次，损失比例 90\.3%
前端适配：柱状图 / 散点图，对比粉丝数量与因果损失规模

### 4）传播全局统计概况

示例：
传播最大深度：5 层
总去重触达人数：650 万人次
分平台触达：微博 606 万、知乎 0\.4 万、头条 35 万、微信 9 万

# 五、模块 3：虚假文本检测

## 5\.1 模块功能

单条新闻文本真伪判定，输出两层结论：

1. 三元分类标签：可信 / 待验证 / 疑似虚假

2. SHAP 词语级解释：标记拉高 / 降低虚假概率的关键词、元数据特征
区分规则：待验证 = 信息不足无法判定；疑似虚假 = 多项特征明确指向谣言

## 5\.2 输出结果明细

### 1）三元真伪判定标签

示例 1：【可信】官方通告，可信度 87%
判定依据：权威认证信源、多平台交叉报道、附带官方引用链接

示例 2：【待验证】网友爆料，可信度 11%
判定依据：独立信源不足，无负面造假特征，无法下定论

示例 3：【疑似虚假】可疑爆料，可信度 2%
判定依据：感叹号密度高、典型谣言关键词、单一信源、无认证账号
前端适配：三色标签 \+ 环形可信度进度条

### 2）SHAP 逐词解释

正负 SHAP 值代表对虚假概率的影响：负值 = 推高虚假概率，正值 = 提升可信度
示例：
虚假关键词（红色高亮）：扩散 SHAP=\-0\.33、！！ SHAP=\-0\.26
可信关键词（绿色高亮）：通报 SHAP=\+0\.19
元数据特征贡献：
粉丝量对数 SHAP=\-1\.61 → 账号粉丝少，虚假风险提升
事件发布时长 SHAP=\-1\.29 → 事件刚爆发信息不全，风险提升
感叹号密度 SHAP=\-0\.87 → 情绪化文本，虚假风险提升
前端适配：原文文字标色高亮、双向横向柱状图展示特征贡献、红绿词云

### 3）五维可信度雷达得分

示例：
信源可信度：53\.1
情感正常度：40\.0
交叉验证度：50\.0
时效风险：53\.0
文本特征得分：43\.9
前端适配：五维雷达图，维度取值 0\~100，数值越高信息越可信

### 4）信息充分度判定

输出：充分 / 一般 / 不足，附带模型不确定度（0\~1，数值越高判断越模糊）

### 5）传播拓扑联合增强判定（依赖模块 2 结果）

文本判定 \+ 传播拓扑特征二次修正，四种修正逻辑：

1. 文本待验证 \+ 拓扑单源头浅层传播 → 升级为疑似虚假

2. 文本待验证 \+ 多源头完整传播链 → 升级为可信

3. 文本判定虚假 \+ 多权威信源传播 → 降级为待验证（证据冲突）

4. 文本判定可信 \+ 单一小众账号扩散 → 降级为待验证

# 六、模块 4：跨事件因果分析

## 6\.1 模块功能

针对多事件并行场景，两套算法交叉验证因果传导关系：

1. 格兰杰因果（线性关系）：历史事件 A 热度能否线性预测事件 B 后续热度

2. 符号化传递熵（非线性关系）：量化事件 A 对事件 B 未来热度的信息约束，无需假设线性关系

## 6\.2 输出结果明细

### 1）因果配对显著性列表

示例：

#### 格兰杰线性因果（p\<0\.05 代表显著）

公共卫生事件 → 明星八卦事件
滞后 5 小时，p=0\.0022（高度显著）
结论：公共卫生负面情绪 5 小时后推高八卦报道量

科技争议事件 → 公共卫生事件
滞后 4 小时，p=0\.0393（显著）

#### 传递熵非线性因果

公共卫生事件 → 科技争议事件
传递熵 TE=0\.0087，p=0\.03（显著）
说明：线性模型无关联，非线性传导存在因果

### 2）事件因果有向拓扑图

示例逻辑：
公共卫生事件 → 明星八卦
科技争议事件 → 公共卫生事件、明星八卦
样式规则：实线 = 格兰杰显著线性因果；虚线 = 传递熵非线性因果；节点大小 = 事件热度；线条粗细 = 显著性强弱（p 值越小线条越粗）
前端适配：有向网络图，展示多事件传导关系

### 3）滞后阶数效应分析

遍历 1\~6 小时滞后窗口，输出各时段 p 值，定位最强因果滞后时间
示例：公共卫生→明星八卦各滞后 p 值
1h：0\.15、2h：0\.07、3h：0\.03、4h：0\.01、5h：0\.002（最强）、6h：0\.01
前端适配：折线图，X 轴 = 滞后小时，Y 轴 = p 值，最低点标注最优滞后窗口

## 6\.3 两种因果算法对比（答辩可用）

|对比维度|格兰杰因果|符号化传递熵|
|---|---|---|
|挖掘关系|线性因果传导|非线性因果传导|
|核心原理|历史 A 提升 B 线性预测精度|A 历史减少 B 未来信息不确定性|
|数据假设|必须满足线性关系|无任何关系假设|
|互补作用|无法识别非线性关联|弥补线性模型遗漏的隐性因果|
|使用策略|双算法同时运行，交叉验证保证可靠性||

# 八、项目依赖安装

## 安装命令

```bash
pip install numpy scikit-learn networkx ruptures joblib
```

## 环境说明

1. 全部算法模块仅依赖 CPU 运行，无需 GPU 显卡

2. 虚假检测模型文件体积仅 85KB，路径`data/fake_detector_model.joblib`

3. 部署时直接复制 data 文件夹至服务根目录即可正常加载模型

---

## 使用说明

1. 全选以上文字，直接复制粘贴到 Word/WPS 文档；

2. 粘贴后自动识别标题层级、表格、代码块；

3. 代码块可设置 Word「等宽字体」，表格一键自动适配页面宽度；

4. 如需生成 docx 文件，复制全部内容粘贴至 WPS 新建空白文档，保存即可。
