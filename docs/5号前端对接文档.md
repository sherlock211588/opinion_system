## 一、总体概览

4号实现了 **4 个算法模块 + 1 个统一报表接口**：

| 模块 | 一句话描述 | 论文集数 |
|------|-----------|:---:|
| 舆情生命周期预测 | 判断事件处于哪个阶段，预测未来走势，提前预警 | 4 篇 |
| 传播溯源 | 画出传播链路图，找出谁在推动传播，量化因果影响力 | 4 篇 |
| 虚假文本检测 | 逐条文章判真假，告诉用户为什么判假 | 5 篇 |
| 跨事件因果分析 | 发现事件之间的因果链：A的情绪变化是否引发了B的热度飙升 | 5 篇 |

## 三、模块1：舆情生命周期预测

### 3.1 这个模块做什么

给一个事件的时序数据，模块1回答四个问题：

- **这个事件现在处于什么阶段？** → 潜伏期 / 成长期 / 高潮期 / 衰退期 / 二次爆发
- **接下来会怎么走？** → 未来 24 小时的预测曲线，带置信区间
- **有没有什么苗头？** → 提前 4~6 小时预警"要炸了"（比等热度涨起来才知道快一步）
- **热度是不是被昼夜周期误导了？** → 自动剥离凌晨低谷，不会把"人都睡了"错判为"热度下降"

### 3.2 能产出什么结果

#### 结果1：阶段标签 + 概率分布

```
当前阶段：高潮期
四阶段概率：潜伏期 2% → 成长期 10% → 高潮期 82% → 衰退期 6%
```
前端可以做一段横向进度条，当前阶段高亮。

#### 结果2：综合热度指数 + 趋势方向

```
综合热度：78（0~100）  趋势：→ 平稳（斜率 -0.02）
```
综合热度不是单纯的报道篇数，而是融合了四个维度的加权结果：
- 报道量（40%）：有多少篇报道
- 情感分歧度（25%）：评论区正面和负面越分歧越好
- 平台扩散度（20%）：覆盖了几个平台
- 互动热度（15%）：每篇的平均互动数

前端可以用一个大号数字展示热度，用 ↑→↓ 箭头展示趋势方向，再用四维雷达图展示 `composite_index_breakdown` 里的四项拆解值。

#### 结果3：未来 24 小时预测曲线

```
+1h:  38.5 篇 [25.0 ~ 52.0]    ← 预测值 + 95%置信下界 & 上界
+2h:  36.2 篇 [22.0 ~ 50.4]
...
+24h: 12.1 篇 [0.0 ~ 28.3]
```

前端画折线图：实线是 `predicted_count`，两条虚线/浅色带宽是 `lower_bound` 和 `upper_bound`。越往后的预测置信带越宽，表示"越远越不确定"。

#### 结果4：阶段切换点（变点检测）

```
07-04 09:00  从潜伏期转入成长期（谷底）
07-04 14:00  热度加速到峰值附近
07-05 20:00  从高潮期转入衰退期（峰值）
```

前端在趋势图上打圆点标记，悬停显示时间和说明文字。

#### 结果5：临界减速预警

```
预警等级：🟠 orange
原因：方差飙升 2.8 倍 + 自相关 0.62，系统不稳定
→ 即使当前报道量不高，极可能在短时间内爆发
```

前端在趋势图上标注预警线和文字。四个等级：
- 🟢 none —— 正常
- 🟡 yellow —— 关注信号，系统开始不稳定
- 🟠 orange —— 预警，逼近临界点
- 🔴 red —— 高风险，极度不稳定

这个预警可以在报道量还没涨的时候就出现——比单纯看趋势曲线提前 4~6 小时。

#### 结果6：二次爆发检测

```
✗ 无二次爆发（或）
✓ 检测到二次爆发（置信度 84%）
  第一轮峰值：07-05 02:00（55 篇）
  谷底：      07-05 18:00（18 篇）
  第二轮峰值：07-06 09:00（48 篇）
  反弹倍率：2.6x
  可能原因：新证据/官方回应/当事人发声引发二次关注
```

前端在趋势图上高亮两轮峰谷，给一个"二次爆发"标签。

#### 结果7：日周期信息

```
日周期已剥离：是
峰值小时：16:00（下午）
低谷小时：03:00（凌晨）
```
如果检测到明显昼夜周期，前端给一个小标签说明"已自动校正昼夜波动"。

## 四、模块2：传播溯源

### 4.1 这个模块做什么

给一个事件的转发关系链，模块2回答四个问题：

- **这件事最开始是谁爆出来的？** → 找到原创源头节点
- **传播路径长什么样？** → 构建整条传播链路图（谁→谁→谁）
- **传播过程中谁最关键？** → 不是看谁粉丝多，是用图算法找出谁在结构上最不可替代
- **如果没有这个人会怎样？** → 每个关键节点的因果影响力："删掉他，会少多少人看到"

### 4.2 能产出什么结果

#### 结果1：传播链路图（力导向图）

```
       ○ 普通网友(200粉, 新浪)           ← 源头
      ╱ ╲
     ╱   ╲
 ○搬运工  ○观察员                       ← 早期扩散
 (5k粉)   (3k粉)
    ╱
   ╱
  ● 大V李教授(50万粉)                   ← 最大节点=PageRank最高
  ╱ ╲           角色：跨圈桥接节点
 ╱   ╲
●官媒  ○地方发布                        ← 官方介入
(500万粉)
╱
○财经频道(20万粉)

节点大小 = PageRank 得分（影响力越大节点越大）
节点颜色 = 角色分类（5种颜色区分普通用户/大V/官媒/源头/桥接）
连线 = 转发关系
```

前端直接用 ECharts / D3.js 的力导向图，绑定 `graph_for_visualization.nodes` + `links` + `categories`。`symbolSize` 控制节点大小，`category` 决定颜色。

#### 结果2：关键节点卡片

```
#1  大V李教授
    角色：跨平台/跨圈桥接节点
    粉丝：50万
    PageRank：0.83 | 介数中心性：1.00 | 出度分：1.00
    综合得分：0.93

#2  新华网官方
    角色：超级传播节点
    粉丝：500万
    PageRank：0.73 | 介数中心性：0.50 | 出度分：0.67
    综合得分：0.63
```

注意：粉丝 500 万的新华网排第二，粉丝 50 万的大V排第一。因为大V的介数中心性=1.0——所有信息流都必须经过他。这就是图算法比单纯看粉丝数更聪明的地方。

前端在传播图旁边放卡片列表，每个关键节点展示四维评分。

#### 结果3：反事实推演（因果影响力排行榜）

```
如果移除 ta，传播规模会发生什么变化？

#1  资讯搬运工（5,000 粉）
    损失：5,755,000 人次（损失 99.9%）
    因为没有 ta，大V和官媒都传播不到

#2  大V李教授（500,000 粉）
    损失：5,750,000 人次（损失 99.9%）

#3  新华网官方（5,000,000 粉）
    损失：5,200,000 人次（损失 90.3%）
```

核心发现：5000 粉的搬运工因果影响力 = 500 万粉的官媒。因为搬运工卡在了传播链的咽喉位置。

前端可以做一张"影响力 vs 粉丝数"的对比表或散点图，让用户直观看到"粉丝多 ≠ 影响力大"。

#### 结果4：传播概况统计

```
传播深度：5 层
总触达：650 万人次（去重后）
  - 微博：606 万
  - 知乎：0.4 万
  - 今日头条：35 万
  - 微信：9 万
数据质量：完整 ✓  /  不完整（有3条转发链缺失）
```

## 五、模块3：虚假文本检测

### 5.1 这个模块做什么

对单篇文章做真假判别，回答两个问题：

- **这条信息可信吗？** → 三元分类：可信 / 待验证 / 疑似虚假
- **为什么判这个结果？** → SHAP 逐词解释：具体哪个词、哪个特征推高了虚假概率

### 5.2 能产出什么结果

#### 结果1：三元判定标签

```
[可信]   官方通告 → 可信度 87%
         信息充分，信源认证+多平台覆盖+含引用链接

[?待验证] 网友爆料 → 可信度 11%
          信息不足，缺乏独立信源验证，暂无负面证据

[!疑似虚假] 可疑消息 → 可信度 2%
            多个虚假信号：感叹号密度过高、含谣言模式词、单信源、无认证
```

"待验证" ≠ "假的"。"待验证"是说"信息不够，没法判断"，而"疑似虚假"是说"有具体证据指向造假"。这个区分是模块3的核心设计——突发事件的独家报道不会被误判为假消息。

#### 结果2：SHAP 逐词解释

```
为什么判"疑似虚假"？

虚假词（推高虚假概率）：
  '扩散'  SHAP=-0.33
  '！！'   SHAP=-0.26

可信词（推高可信概率）：
  '通报'  SHAP=+0.19

元数据特征贡献：
  log_follower_count  SHAP=-1.61  →  粉丝太少 → 虚假信号
  hours_since_event   SHAP=-1.29  →  刚爆发信息不足 → 虚假信号
  exclamation_density SHAP=-0.87  →  感叹号太多 → 虚假信号
```

前端拿到 `shap_explanation.top_words` 后可以：
- 在原文上高亮：虚假词标红色，可信词标绿色
- 在判定结果旁边展示"为什么这样判"的卡片
- 词云：虚假信号词用红色，可信信号词用绿色

元数据特征贡献可以做一条横向柱状图：正值向右（绿色，推送可信），负值向左（红色，推送虚假）。

#### 结果3：五维雷达得分

```
信源可信度：      53.1
情感正常度：      40.0
交叉验证度：      50.0
时效风险：        53.0
文本特征得分：    43.9
```

前端画五维雷达图，每个维度 0~100，越高越可信。

#### 结果4：信息充分度

```
信息充分度：一般
不确定度：0.57（模型自己也不太确定）
```

前端可以用这两个字段展示"这个判断有多可靠"的辅助信息。

#### 结果5：传播拓扑增强（可选，需要模块2先跑）

如果传播图已经建好，可以把图拓扑特征传给虚假检测做联合判断：

```
文本模型判：待验证
传播拓扑判：单源头+深链+单平台 → 假新闻模式
最终判定：疑似虚假（拓扑增强）

原因："文本模型犹豫，但传播拓扑强烈指向虚假→升级为疑似虚假"
```

四个可能的增强效果：
- 文本判待验证 + 拓扑判假 → **升级为疑似虚假**
- 文本判待验证 + 拓扑判真 → **升级为可信**
- 文本判假 + 拓扑判真 → **降级为待验证**（证据矛盾）
- 文本判真 + 拓扑判假 → **降级为待验证**

### 5.3 输入 / 输出

**初始化**（只需一次）：

```python
from time_series.fake_detection import get_or_train_model, FakeDetector

```

| `sentiment_intensity` | float | 3号 | `0.5` | 情感强度 (0=中性, 1=极端) |

**批量判别**（推荐 — Pipeline 自动做字段映射）：

```python
results = pipe.article_check(articles)
# 或
report = pipe.event_report(event_data, articles, propagation_nodes)
```

**`articles` 输入格式**（Pipeline 的 `article_check` 和 `event_report` 都用）：

```python

    {
        "id": "1",              # 文章唯一ID
        "title": "官方预警",     # 标题
        "cleaned_text": "...",  # 2号清洗后的纯文本 (优先) 或 "text": "..."
        "is_verified": True,    # 1号 → 映射到 source_verified
        "follower_count": 5000000,  # 1号 → 映射到 source_followers
        "forward_count": 50,        # 1号 → 映射到 similar_report_count
        "hours_since_event": 4.0,   # 3号 → 映射到 hours_since_event_start
        "sentiment_intensity": 0.1, # 3号 → 直传
        "source": "央视新闻",       # 可选，前端展示用
        "publish_time": "2026-07-06 08:00",  # 可选，前端展示用
    },
    # ...
]
```
> Pipeline 内部做字段映射（如 `is_verified`→`source_verified`）。缺失字段自动用默认值，不会崩溃。

## 六、模块4：跨事件因果分析

### 6.1 这个模块做什么

系统里有多个事件同时发生——事件A的情绪变化，是否在几小时后引起了事件B的报道量变化？模块4跑两条因果检测线，交叉验证：

- **格兰杰因果（线性）：** 事件A的过去值是否能显著提升对事件B当前值的线性预测精度？
- **符号化传递熵（非线性）：** 知道事件A的过去，能减少事件B未来多少不确定性？（不需要假设线性关系）

### 6.2 能产出什么结果

#### 结果1：因果对列表

```
格兰杰因果（线性）
─────────────────────────────────
公共卫生事件 → 明星八卦事件
  lag = 5 小时  |  p = 0.0022  *** 显著 ***
  → 公共卫生的负面情绪在 5 小时后推高了明星八卦的报道量

科技争议事件 → 公共卫生事件
  lag = 4 小时  |  p = 0.0393  *** 显著 ***

传递熵（非线性）
─────────────────────────────────
公共卫生事件 → 科技争议事件
  TE = 0.0087  |  p = 0.03  *** 显著 ***
  → 格兰杰没发现，但传递熵发现了非线性因果
```

#### 结果2：事件因果有向图

```
  公共卫生 ──→ 明星八卦
      ↑
  科技争议 ──→ （也指向明星八卦）

  实线 = 格兰杰显著  虚线 = 传递熵显著
```

前端可以做一张"事件关系图"：每个节点是一个事件，节点大小按热度，有向边表示因果方向，边的粗细按显著性（p 值越小越粗），实线=线性因果，虚线=非线性因果。

#### 结果3：滞后效应分析

对每对显著因果，展示各滞后阶数（1~6小时）的 p 值变化：

```
公共卫生 → 明星八卦的 p 值随滞后变化：
  1h:  p=0.15
  2h:  p=0.07
  3h:  p=0.03
  4h:  p=0.01
  5h:  p=0.002  ← 最显著
  6h:  p=0.01
```
前端画一条"p 值 × 滞后小时"的折线，最低点就是最强因果滞后。

### 6.3 两种方法的区别（答辩用）

| | 格兰杰因果 | 符号化传递熵 |
|------|:---:|:---:|
| 发现什么 | 线性因果关系 | 非线性因果关系 |
| 原理 | 看 driver 的历史值能否提升 target 的线性预测 | 看 driver 的过去能减少 target 未来多少信息不确定性 |
| 假设 | 线性关系 | 无假设 |
| 互补性 | 漏掉非线性的 | 补上非线性的 |
| 两个都跑 | → 交叉验证，更可靠 | |
