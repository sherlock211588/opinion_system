## <a id="1号"></a>二、1号（爬虫）—— 需要你输出 6 个字段

### 1号需要输出的字段

| 字段 | 类型 | 说明 | 示例 | 重要性 |
|------|------|------|------|:---:|
| `author_name` | **str** | 发帖账号名 | `"北京日报"` | 🟡 |
| `follower_count` | **int** | 该账号的粉丝数 | `5000000` | 🟡 |
| `is_verified` | **bool** | 是否认证用户 | `true` | 🟡 |
| `forward_count` | **int** | 被转发了多少次 | `320` | 🟡 |
| `parent_article_id` | **str 或 null** | 转发自哪篇文章。`null`=原创首发 | `null` 或 `"1"` | 🔴 |
| `content_summary` | **str** | 原文前50字摘要 | `"北京市气象台升级..."` | 🟢 |

> 🔴 `parent_article_id` 是整个传播溯源功能的命脉。没有它，传播链路图、关键节点、反事实推演全部失效。**这个字段必须准。**

### 1号输出示例

```json
// 例子1：原创文章
{
  "id":                  "1",
  "author_name":         "北京日报",
  "follower_count":      5000000,
  "is_verified":         true,
  "forward_count":       320,
  "parent_article_id":   null,
  "content_summary":     "北京市气象台升级发布高温橙色预警"
}

// 例子2：转发文章（转自文章1）
{
  "id":                  "8",
  "author_name":         "某网友",
  "follower_count":      200,
  "is_verified":         false,
  "forward_count":       15,
  "parent_article_id":   "1",
  "content_summary":     "今天确实太热了，大家注意防暑"
}

// 例子3：再次转发（转自文章8）
{
  "id":                  "15",
  "author_name":         "大V张教授",
  "follower_count":      500000,
  "is_verified":         true,
  "forward_count":       5000,
  "parent_article_id":   "8",
  "content_summary":     "高温预警值得重视，转发提醒"
}
```

> 这三条数据连起来就是一条传播链：`id=1(原创) → id=8(转发) → id=15(大V转发)`，我拿着直接画传播图。

## <a id="3号"></a>三、3号（NLP）—— 需要你输出 6 个字段

### 3号需要输出的字段

| 字段 | 类型 | 说明 | 示例 | 重要性 |
|------|------|------|------|:---:|
| `event_id` | **str** | 事件唯一ID，同一事件的文章共用 | `"EVT_20260706_001"` | 🔴 |
| `event_title` | **str** | 事件名称 | `"高温天气相关报道"` | 🟡 |
| `category` | **str** | 五选一：`社会民生`/`科技互联网`/`娱乐八卦`/`财经商业`/`国际时政` | `"社会民生"` | 🟡 |
| `sentiment_label` | **str** | 三选一：`"正面"`/`"负面"`/`"中立"` | `"中立"` | 🟡 |
| `sentiment_intensity` | **float** | 情感强度，必须 0~1 的小数 | `0.3` | 🔴 |
| `sentiment_positive_ratio` | **float** | 正面情感占比 0~1 | `0.40` | 🟡 |
| `sentiment_negative_ratio` | **float** | 负面情感占比 0~1 | `0.10` | 🟡 |

> 🔴 `event_id` 决定我能否按事件聚合——同一事件的所有文章必须用同一个 event_id。
> 🔴 `sentiment_intensity` 我虚假检测模型的 9 维特征之一，缺了准确率降 5~8%。必须是 0~1 的浮点数（`0.82`），不要填 `82`。

### 3号输出示例

```json
// 文章1：同一事件的第一条
{
  "id":                      "1",
  "title":                   "北京发布高温橙色预警",
  "source":                  "新浪新闻",
  "publish_time":            "2026-07-06 10:30:00",
  "cleaned_text":            "北京发布高温橙色预警。今日北京市气象台...",

  "event_id":                "EVT_20260706_001",
  "event_title":             "北京高温天气",
  "category":                "社会民生",
  "sentiment_label":         "中立",
  "sentiment_intensity":     0.3,
  "sentiment_positive_ratio": 0.40,
  "sentiment_negative_ratio": 0.10
}

// 文章2：同一事件的另一篇文章（event_id 相同）
{
  "id":                      "5",
  "title":                   "高温天气下的户外工作者",
  "source":                  "澎湃新闻",
  "publish_time":            "2026-07-06 14:20:00",
  "cleaned_text":            "连日高温，户外工作者面临严峻考验...",

  "event_id":                "EVT_20260706_001",
  "event_title":             "北京高温天气",
  "category":                "社会民生",
  "sentiment_label":         "负面",
  "sentiment_intensity":     0.7,
  "sentiment_positive_ratio": 0.15,
  "sentiment_negative_ratio": 0.75
}
```

## <a id="规范"></a>四、公共格式规范（所有人必须遵守）

| 规则 | 正确 | 错误 |
|------|------|------|
| 字段名 | `publish_time` | ~~`publishTime`~~ ~~`PublishTime`~~ |
| 时间 | `"2026-07-06 10:30:00"` | ~~`"2026/07/06"`~~ |
| null | `"parent_article_id": null` | ~~`""`~~ ~~`"null"`~~ |
| bool | `true` / `false` | ~~`1`/`0`~~ ~~`"是"`/`"否"`~~ |
| float | `0.82` | ~~`82`~~ ~~`"82%"`~~ |
| 文件格式 | `.jsonl`（每行一条 JSON） | ~~一个数组包所有数据~~ |
